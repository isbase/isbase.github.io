vue项目服务器部署,最主要是得速度

Author: John Doe Date: May 10, 2020  0:00:00 Category: wechat,h5

  • 刚写完了一个钉钉企业内应用,之后还没部署.以前这些事都是别人来搞的,今天貌似需要我这个懒癌动手了

  • 迫于后端只给服务器,所以环境自己搭建,

  • 看了一下子服务器没有任何web环境

  • 所以现在就需要解决 2个问题,

    1. web 环境配置
    2. 解决跨域的代理(其实都是一件事)

开始吧

  • 想法就是快准狠. 最好一键安装,一键部署
  • 先看看这机器啥配置,啥系统
  • 宝塔 CentOS

等待安装完毕,之后登陆

  1. 软件商店安装 nginx

  2. 网页菜单新建站点

  3. 绑域名 && 解析

  4. npm run build 上传包

  5. 访问正常….

  6. nginx 反向代理

    1. 宝塔内打开网站,设置.

    2. 反向代理设置,随便添加一个.名称写目录,比如我写的 api, 其实是用来区分,本地文件跟 api 接口的,我再 build 的是时候设置 axios 的 baseURL = VUE_APP_BASE_API = ‘./api’ 所以….

    1. 新建反向代理之后,点击反向代理的配置文件. 检查一下.

    2. location /api
 {
  proxy_pass http://123.33.33.33:8792/ss-mobile;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header REMOTE-HOST $remote_addr;
  ...
}

    3. 到这里就好了

    4. 如果没有基础环境,这样已经很快了,

    5. 关于部署,其实还可以 git + WebHook , 以后直接 push 就好了.自动更新.
